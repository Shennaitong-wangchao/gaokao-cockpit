# Product Spec / 产品说明

This is the public product specification for Gaokao Cockpit. It describes the current open-source product direction, MVP scope, and non-goals. It intentionally avoids private learner data, school-specific assumptions, and unpublished roadmap commitments.

## 产品定位

Gaokao Cockpit / 高考驾驶舱 是一个 local-first 的个人学习操作系统，面向需要长期备考、每日执行、错题诊断和复盘闭环的学习者。

它不是教育平台、课程平台或题库平台，而是一个每天打开后能推动用户进入学习闭环的个人驾驶舱。

核心定位：

```text
帮助用户完成：
启动 -> 计划 -> 专注 -> 做题记录 -> 错题手术 -> Prompt 调用 -> 每日复盘 -> 周复盘 -> 明日继续
```

第一版优先服务“本地优先、每天可打开、流程稳定、减少学习执行摩擦”。

## 目标用户

主要用户：

- 中国高中生或其他以考试备考为核心场景的学习者。
- 有明确长期目标，需要把目标拆成每天能执行的动作。
- 每天会使用 AI 辅助学习。
- 已经有教材、试卷、练习册、错题、笔记、AI 对话等多个学习来源。
- 需要一个稳定的个人学习流程，而不是更多零散工具。

用户特征：

- 学习目标很高，但每天状态会波动。
- 需要把长期目标拆成当天可执行动作。
- 需要低输入成本，不愿把时间耗在维护系统上。
- 需要 AI 提供讲解、追问、变式、诊断，而不是直接给答案。

## 核心使用场景

1. 早上打开 App，快速确定今日状态、主攻科目和 Top 任务。
2. 学习前创建或选择任务，进入专注计时。
3. 做题后记录结果，尤其是值得复盘的错题。
4. 遇到难题或错题时，从 Prompt 仓库生成提示词并复制给 AI。
5. 晚上完成每日复盘，确认今天完成了什么、最大问题是什么、明天第一步是什么。
6. 周末完成周复盘，查看科目投入、错题类型、下周重点。
7. 状态差时进入降载流程，只完成保底任务，避免整天断线。

## 每日学习闭环

```text
[启动]
  记录起床时间、当前状态、今日主攻科目

[计划]
  设置 Top 任务、保底任务、奖励任务、明日第一步

[专注]
  选择任务 -> 开始计时 -> 记录分心与完成度

[做题记录]
  记录实际用时、完成结果、产出备注

[错题手术]
  拆解错题：错因、根因、题目信号、正确模型、变式任务

[Prompt 调用]
  根据场景生成 Prompt -> 复制到剪贴板 -> 粘贴到 AI 工具

[每日复盘]
  总结完成、未完成、最大问题、最佳错题、明日第一步

[周复盘]
  汇总学习时间、科目分布、错题分布、下周重点
```

闭环的关键不是记录得多，而是每天都能回到下一步。

## 核心模块

### 今日驾驶舱

今日驾驶舱是 App 的第一屏。

它回答三个问题：

- 今天我的状态如何？
- 今天最重要的学习动作是什么？
- 如果状态差，最低要完成什么？

核心内容：

- 日期。
- 起床时间。
- 当前状态评分。
- 今日主攻科目。
- Top 任务。
- 保底任务。
- 奖励任务。
- 明日第一步。
- 今日进度摘要。

### 学习任务

学习任务是每日执行单元。

每个任务必须足够具体，最好能在 25 到 90 分钟内完成，例如：

- 数学：完成导数压轴题 6 题，并记录错因。
- 英语：精读一篇阅读理解，整理生词和长难句。
- 物理：复盘电磁感应错题 3 道，写出模型触发信号。

任务不是愿望清单，而是可以开始、计时、完成、复盘的动作。

### 专注计时

专注记录绑定学习任务。

它不追求复杂番茄钟，而是记录：

- 计划学多久。
- 实际学多久。
- 分心次数。
- 完成评分。
- 本轮产出。
- 下一步。

### 错题手术

错题不是收藏，而是手术。

一条错题记录必须尽量回答：

- 我为什么错？
- 哪个知识点或模型没有启动？
- 题目里哪个信号应该触发正确方法？
- 正确模型是什么？
- 下一次如何复练？

错题手术的价值在于“形成下次能做对的机制”，不是保存题目截图。

### Prompt 仓库

Prompt 仓库内置常用学习教练模板。

第一版不接 AI API，只提供：

- 选择模板。
- 填写少量变量。
- 生成完整 Prompt。
- 一键复制到剪贴板。

AI 的定位是教练组：

- 错题外科医生。
- 教材预习教练。
- 课后整理教练。
- 变式题教练。
- 单元诊断教练。
- 每日复盘教练。
- 周复盘教练。

### 学习资料

学习资料用于建立轻量索引，不做完整知识库。

可记录：

- 教材章节。
- 试卷。
- 练习册。
- 讲义。
- 视频。
- 网页。
- 本地文件路径。

第一版只保存标题、科目、章节、类型、URI、状态和备注。

### 每日复盘

每日复盘用于收束当天。

必须短、稳定、可完成：

- 今天完成了什么？
- 什么没完成？
- 最大问题是什么？
- 今天最值得手术的一道错题是什么？
- 晚上状态评分是多少？
- 明天第一步是什么？

### 周复盘

周复盘用于发现结构性问题。

它关注：

- 总学习时长。
- 各科投入。
- 完成任务数。
- 错题数量。
- 错题类型分布。
- 本周关键问题。
- 下周主攻方向。

## MVP 功能范围

MVP 必须包含：

- 本地创建和查看今日计划。
- 本地创建和管理学习任务。
- 基础专注计时与记录。
- 错题手术记录。
- Prompt 模板生成与复制。
- 学习资料轻量索引。
- 每日复盘。
- 周复盘。
- 所有数据本地保存。

MVP 可以简化：

- 统计图表可以先用文字和简单列表。
- 周复盘可以先手动生成，不做复杂自动分析。
- Prompt 变量可以用普通文本输入。
- 错题图片可以先保存本地路径，不做 OCR。

## 非目标

第一版明确不做：

- 账号登录。
- 云同步。
- 支付。
- 社交。
- 排行榜。
- AI API 调用。
- 自动解题。
- 自动批改。
- 自动规划整周学习。
- 大型题库。
- 学校班级管理。
- 家长端。
- 老师端。
- 多人协作。
- macOS、Web、后端服务。

## 关键设计原则

1. 每天能打开，而不是做得很复杂。
2. 每个页面都要服务学习闭环。
3. 减少输入成本。
4. 不把用户拖进系统设计，而是把用户推回学习。
5. 状态差时也要有保底流程。
6. 错题不是收藏，而是手术。
7. AI 不是答案机，而是教练组。
8. 第一版不追求全自动，只追求流程稳定。
9. 不做社交，不做排行榜，不做账号，不做云同步。
10. 本地优先，学习闭环优先，隐私边界优先。

## 成功标准

### 使用成功标准

- 用户每天早上 1 分钟内能完成启动。
- 用户能在 3 次点击以内进入当前学习任务。
- 用户能在学习后 2 分钟内完成任务记录。
- 用户能在 5 分钟内完成一条高质量错题手术。
- 用户能在 3 分钟内完成每日复盘。
- 用户状态差时仍能完成至少一个保底任务。

### 产品成功标准

- 连续 7 天每天打开。
- 每天至少记录 1 个学习任务。
- 每周至少完成 3 条错题手术。
- 每周至少完成 1 次周复盘。
- 用户能清楚说出“明天第一步做什么”。

### MVP 交付成功标准

- 无账号、无网络也能完整使用。
- 核心数据能本地持久化。
- Prompt 能生成并复制。
- 页面之间能形成完整闭环。
- 没有引入会拖慢第一版的复杂系统。

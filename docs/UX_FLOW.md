# UX Flow / 用户体验流程

This document describes the intended user journeys and screen-level information architecture for the open-source app. Some ASCII sketches are historical low-fidelity references; current implementation details may differ, and code should be treated as the source of truth for exact UI behavior.

本文档描述 Gaokao Cockpit 第一版的主要体验流程和页面草图。草图为低保真 ASCII，只表达信息结构，不代表最终 UI。

## 总体导航

当前实现使用底部 Tab。重点不是导航样式本身，而是每天能快速进入学习闭环。

```text
┌───────────────────────────────┐
│ Today 今日                     │
├───────────────────────────────┤
│ Tasks 任务                     │
│ Focus 专注                     │
│ Mistakes 错题                  │
│ Prompts Prompt                 │
│ Reviews 复盘                   │
└───────────────────────────────┘
```

核心页面：

- TodayCockpitView 今日驾驶舱
- TaskListView 学习任务
- FocusSessionView 专注计时
- MistakeSurgeryView 错题手术
- PromptLibraryView Prompt 仓库
- DailyReviewView 每日复盘
- WeeklyReviewView 周复盘

## 第一次打开 App 的流程

目标：不做复杂 onboarding，让用户尽快进入今天。

```text
首次打开
  -> 显示一句定位
  -> 可选填写考试年份
  -> 可选填写目标分数
  -> 进入今日驾驶舱
```

页面草图：

```text
┌───────────────────────────────┐
│ 高考驾驶舱                     │
│ 每天启动、专注、错题、复盘。    │
│                               │
│ 考试年份  [可选]                │
│ 目标分数  [700]                │
│                               │
│ [进入今日]                     │
└───────────────────────────────┘
```

MVP 可以不保存复杂用户档案，只保存最少偏好。也可以先跳过 onboarding，直接进入今日驾驶舱。

## 每天早上启动流程

目标：1 分钟内完成启动。

```text
打开 App
  -> TodayCockpitView
  -> 如果今天没有 DayPlan，创建今日计划
  -> 记录 wakeTime / stateScore / mainSubject
  -> 填 1 到 3 个 Top 任务
  -> 填 1 到 2 个保底任务
  -> 开始第一个任务
```

## 专注计时流程

目标：专注计时服务学习，不做过度复杂番茄钟。

```text

  -> 结束
```

## 错题手术流程

目标：把错题变成下次能做对的机制。

```text
MistakeSurgeryView
  -> 新建错题
  -> 录入题目文字或图片路径
  -> 填我的解法和正确解法
  -> 选择错误类型
  -> 写根因、题目信号、正确模型
  -> 写变式任务和提醒
  -> 可生成错题手术 Prompt
```

## Prompt 生成流程

目标：不接 AI API，只做一键生成和复制，让用户继续使用自己熟悉的 AI 工具。

```text

  -> 填少量变量
  -> 生成 Prompt
  -> 复制到剪贴板
```

```text

  -> 填完成总结、未完成原因、最大问题
  -> 选择今日最佳错题
```

## 状态差时的降载流程

目标：状态差时不追求完美，只保住学习链条不断。

触发条件可以是：

- 早上 stateScore <= 4。
- 当天连续拖延。
- 专注中分心次数明显过高。
- 晚上发现 Top 任务已无法完成。

降载策略：

```text
状态差
  -> 隐藏奖励任务
  -> Top 任务缩小
  -> 只显示保底任务
  -> 推荐 15 到 25 分钟短专注
  -> 只要求完成 1 条错题手术或 1 个最小任务
  -> 晚上仍完成每日复盘
```

```text
├───────────────────────────────┤

│ [ ] 英语单词 20 个              │
```

## 体验边界

第一版体验必须克制：

- 不在首页堆过多统计。
- 不要求用户维护复杂知识库。
- 不把每个错题都变成必填大表单。
- 不把 Prompt 做成聊天产品。
- 不用社交激励替代真实学习动作。
- 不让用户花很多时间美化任务系统。
